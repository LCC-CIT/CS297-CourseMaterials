Creating and Setting up a project

Assumptions

  • You already have an account on Azure DevOps.
  • You are  a member of an organization.
    • Your access level is Basic.

Create a Project

  1. In the left-hand menu, click on Projects
  2. Click on New team project...
    • Enter a project name and description
    • Select either public or private--you can have up to 5 members in a private project
    • Select advanced
      • Under version control, select Git
      • Under work item process, select Scrum (reccomended) or Agile
        Information about the selections
        • Agile gives you these work items:
          • Epic
            • Feature
              • User Story or Bug
                • Task
          • Issue
        • Scrum giives you these work items
          • Epic
            • Feature
              • Product backlog item or Bug
                • Task
          • Impediment
  3. Click on Create Project

Add Members to the Organization

  1. Click on the Azure DevOps logo (at the top left of the screen)
    • Click on Organization settings (at the bottom left of the screen)
      • Click on Users
      • To add new team members, click on + Add new users
        • Users: Enter a user's e-mail address.
        • Access level: Choose an option, Basic is usually what you want.
          • Basic--user can access all features. Up to 5 are allowed on the free plan
          • Stakeholder--limited access to features (mainly viewing). An unlimited number are allowed on the free plan.
        • Projects--select one or more projects.
        • Azure DevOps Groups--select the appropriate group, usually Project Contributors
          • Project Readers--read-only access.
          • Project Contributrs--able to contribute fully to the project.
          • Project Administrators--able to configure project resources.
        • Check the send email invites box
        • Click the Add button.

Add Members to the Project Team

In the left-hand menu, click on Project settings (at the bottom)

  1. Under General, selct Teams
    • You shoudl see a team created for your project, for example, DemoProejct Team, click on it.
      • Under Administrators (in the center column), click on +Add
        • Enter the name of the scrum master in the search, when they are found select them, then click on Save chagnes.
          I reccomend just making the Scrum Master for the current sprint the administrator, but you can add more administrators if you wish.
      • Under Members (in the right-hand pane) click on +Add and enter the rest of the team members.
        • Give each team member the access level of Stakeholder.
  2. Under Project Settings, Boards, select Tem configuration
    • Select the backlogs you want to maintain. Do you want both Epics and Features, or just Features. I reccmend the default, which is just Features and Backlog Items
    • Select your work days. This is up to your team.
    • Under Working with bugs, I reccomend you leave the default setting, Bugs are maintained with requirements.

Create and Manage User Stories

Create Work Items

These instructions assume you are using the Scrum process, the Work Item Types are:

  • Epic
    • Feature
      • Product Backlog Item (aka User Story)
        • Task
  • Bug
    • Task
  • Impediment (aka issue)
  1. Click on Boards, then Boards (in the left-hand menu)
  2. Select the type of board, Feature or Product backlog item--depending on what kind of item you want to add.
  3. Click on +New Item, then give it a name like, "Create user profile"
    • Click on the item and add a description. For example, "Users will be able to create a profile so that they can share information about themselves with other users."
    • Set the priority.
    • Enter the time estimate in the effort field
    • If it's a feature, then click on Related work and add one or more Product backlog items (user stories)

Move Bakclog Items (User Stories) to Sprints

  1. Click on Boards, then Backlog (in the left-hand menu)
  2. Click on the view options (slider controls icon) and select the planning side pane
  3. Drag and drop Backlog Items into sprints
  4. Assign a developer to each user story and enter an estimated effort (if you haen't already).

See: Assign Backlog Items to a Sprint

Create Tasks in a Sprint

  1. Click on Boards, then Sprints (in the left-hand menu)
  2. For any Backlog Item (user story), to it's right, click on + New Item, to add one or more tasks
    The tasks are what move across the swim-lanes, not the Backlog Item.
  3. Assign a developer to each task (normally the same one that is assigned to the Backlog Item (User Story).
  4. As development progresses, update the Remaining Work and Completed Work fields.
    The completed work in the tasks that are done is what I will use to calculate the time you've spent working on the project.

For reference, read the tutorial--Track user stories, tasks, and features .

Notes:
You can assign Product backlog items and tasks to a Sprint, but not Epics or Features.
Only tasks will move across the task board.

For more details, read the documentation: Sprints (Scrum) .
