Creating and Setting up a project

Assumptions

  • You already have an account on Azure DevOps
  • You are  a member of an organization
    • Your access level is Basic.

Creating a Project

  1. In the left-hand menu, click on Projects
  2. Click on New team project...
    • Enter a project name and description
    • Select either public or private--you can have up to 5 members in a private project
    • Select advanced
      • Under version control, select Git
      • Under work item process, select Scrum (reccomended) or Agile
        Information about the selections
        • Agile gives you these work items:
          • Epic
            • Feature
              • User Story or Bug
                • Task
          • Issue
        • Scrum giives you these work items
          • Epic
            • Feature
              • Product backlog item or Bug
                • Task
          • Impediment
  3. Click on Create Project

Configuring a Project

In the left-hand menu, click on Project settings (at the bottom)

  1. Under General, selct Teams
    • You shoudl see a team created for your projec, for example, DemoProejct Team, click on it.
      • Under Administrators (in the center column), click on +Add
        • Enter the name of the scrum master in the search, when they are found select them, then click on Save chagnes.
          I reccomend just making the Scrum Master for the current sprint the administrator, but you can add more administrators if you wish.
      • Under Members (in the right-hand pane) click on +Add and enter the rest of the team members.
  2. Under Project Settings, Boards, select Tem configuration
    • Select the backlogs you want to maintain. Do you want both Epics and Features, or just Features. I reccmend the default, which is just Features and Backlog Items
    • Select your work days. This is up to your team.
    • Under Working with bugs, I reccomend you leave the default setting, Bugs are maintained with requirements.

Creating Work Items

  1. Click on Boards, then Work Items (in the left-hand menu)
    • Next to +New Work Item, select the type, typically Feature, and click on it.then give it a name like, "User registration"
      The terminology here doesn't include User Story, but think of a Feature as a User Story and write it's description in that style.
    • Add a description. For example, "Users will be able to create a profile so that they can share information about themselves with other users."
